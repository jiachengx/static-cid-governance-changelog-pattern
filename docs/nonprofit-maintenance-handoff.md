# Nonprofit Maintenance Handoff

Nonprofit maintainers often inherit systems without full historical context. A governance changelog helps future maintainers understand why certain files, rules, or release checks must not be casually removed.

A handoff-oriented changelog should record:

- release purpose;
- governance reason;
- public/private boundary assumptions;
- validation rules added or removed;
- migration requirements;
- files or directories that future maintainers should not delete;
- known limitations.

This makes the changelog useful as a lightweight continuity document, not only a development log.
