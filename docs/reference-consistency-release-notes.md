# Reference Consistency Release Notes

Reference behavior can change silently if it is not documented.

A governance changelog should record whether a release affects:

- stable references;
- latest-tracking references;
- pinned snapshot references;
- source-reference metadata;
- stale-reference detection;
- projection regeneration;
- release-gate validation.

A useful release note states not only that a reference bug was fixed, but whether existing data must be migrated or regenerated.
